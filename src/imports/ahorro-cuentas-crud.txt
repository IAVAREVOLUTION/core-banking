5. Reglas institucionales del Modo Editar
No eliminar campos del JSON.

No reconstruir el JSON desde cero.

No modificar id.

No modificar linea_produc (siempre “CAPTACION”).

No modificar tipo_produc (siempre “Ahorro”).

Respetar llaves foráneas.

Mantener estructura padre + nodos JSON.

3. Construcción del JSON parcial
El sistema debe construir un JSON solo con los campos modificados, por ejemplo:

json
{
  "cliente": {
    "claveCliente": "CL-001",
    "nombreCompleto": "Juan Pérez"
  },
  "producto": {
    "claveProducto": "AHO-001",
    "nombreProducto": "Ahorro Tradicional",
    "tasa": 5.5,
    "montoMinimo": 1000
  },
  "metadatos": {
    "ultimaActualizacion": "2026-02-27"
  }
}
Reglas:

No eliminar nodos.

No enviar campos no modificados.

No reconstruir el JSON completo.

Reglas del MERGE:
data SIEMPRE va a la izquierda.

El JSON parcial va a la derecha.

Los campos no enviados se conservan.

No se borra nada del JSON existente.

5. Validaciones obligatorias
no_sol no puede estar vacío.

no_cuenta no puede estar vacío.

cliente_id debe existir en "J_CLIENTES".

producto_id debe existir en "J_PRODUCTOS".

linea_produc debe seguir siendo “CAPTACION”.

tipo_produc debe seguir siendo “Ahorro”.

No permitir guardar sin JSON.

El registro queda actualizado:
